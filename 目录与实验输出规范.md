# 目录与实验输出规范

## 固定目录

| 路径 | 用途 | 清理策略 |
| --- | --- | --- |
| `datasets/` | 本地数据集，只读输入 | 不手动清理 |
| `Qwen3-32B/` | 本地模型权重 | 不手动清理 |
| `harness-sii/` | 官方 harness、工具、搜索/浏览适配 | 只保留代码、README、必要提交文件 |
| `实训尝试codex/` | 各探索方向代码和共享 adapter | 只保留代码、配置、文档 |
| `runs/dev/` | smoke、小样本、调试运行 | 可随时删除 |
| `runs/eval/` | 正式验证集/数据集跑分输出 | 保留最新有效结果；旧结果归档或删除 |
| `runs/benchmark/` | 最终 benchmark/提交前运行 | 保留 |
| `runs/archive/` | 临时归档旧实验摘要 | 只放仍有对照价值的旧结果 |
| `reports/` | 汇总表、对比分析、分数记录 | 保留 |
| `submissions/` | 最终提交包、CSV/JSON/ZIP | 保留 |

## 运行输出结构

后续所有新实验统一写到 `runs/` 下：

```text
runs/
  dev/<run_name>/<dataset>/<track>/
  eval/<run_name>/<dataset>/<track>/
  benchmark/<run_name>/<dataset>/<track>/
```

每个 track 目录只保留这些默认产物：

- `run.log`
- `summary.json`
- `results.jsonl`
- 必要时才保留 `trajectories.jsonl`

`trajectories.jsonl` 通常体积最大。调试完成后，如果只需要分数对照，应删除轨迹文件，保留 `summary.json` 和 `results.jsonl`。

## 推荐命令

小样本 smoke：

```bash
cd /inspire/qb-ilm2/project/26summer-camp-01/26210300/实训尝试codex
conda run -n pegp python -m shared_sii_adapter.run_dataset \
  --input ../datasets/2wiki.jsonl \
  --track track_e \
  --output-dir ../runs/dev/tool7_smoke \
  --run-mode dev \
  --disable-evolution-updates \
  --limit 20 \
  --concurrency 50
```

多 track 验证：

```bash
cd /inspire/qb-ilm2/project/26summer-camp-01/26210300/实训尝试codex
conda run -n pegp python scripts/run_all_datasets.py \
  --output-dir ../runs/eval/tool7_c50 \
  --datasets all \
  --tracks all \
  --limit 50 \
  --concurrency 50
```

## 清理规则

可以直接删除：

- `__pycache__/`
- `.pytest_cache/`
- `.ipynb_checkpoints/`
- `.DS_Store`
- `runs/dev/*`
- 已确认不再复盘的 `trajectories.jsonl`
- 旧的 smoke 目录，如 `experiments_smoke*`

谨慎删除：

- `runs/eval/*`
- `validation_full/`
- `validation_runs/`
- `validation_submit_bundle/`
- `submissions/`

不要删除：

- `datasets/`
- `Qwen3-32B/`
- 当前代码目录
- 最新提交包和最终 benchmark 输出
