# 验证集自进化运行说明

## 合规边界

- Harness 基座模型固定使用 `Qwen3.5-9B`。
- `Qwen3-32B` 只用于 LLM-as-a-Judge、反思和记忆组织，不作为答题基座模型。
- Judge 输入不包含 `answer` / gold / ground truth。
- SimpleVQA 的 gold answer 只在每个样本推理结束后由本地脚本离线评分使用。
- 2Wiki 当前文件的 `answer` 字段为空，因此只能产出预测和轨迹，不能本地计算准确率。
- 打榜数据集不参与该验证集自进化优化流程。

## 已适配的数据集

- `datasets/2wiki.jsonl`
- `datasets/simpleVQA/SimpleVQA.jsonl`

SimpleVQA 会同时传入本地图片 base64 和 `image_url`。模型可以直接看图，也可以在开启工具时调用图搜。

## 关键脚本

- `harness-sii/run_validation.py`
- `harness-sii/run_one_case.py`
- `harness-sii/task_runner.py`

输出格式遵循课题 PDF：

```json
{"index": 0, "instruction": "...", "image": "...", "answer": "...", "pred": "..."}
```

轨迹会额外汇总到同目录的 `*_trajectories.jsonl`。

## 当前完整运行结果

完整结果在 `harness-sii/validation_full`：

- 2Wiki baseline：100 条，57 条非空预测；该文件 gold answer 为空，无法本地算准确率。
- 2Wiki evolved：100 条，45 条非空预测；100 条均有 32B Judge，45 条 Judge pass；该文件 gold answer 为空，无法本地算准确率。
- SimpleVQA baseline：99 条，63 条非空预测；别名归一后 `31/99 = 31.31%`。
- SimpleVQA evolved：99 条，51 条非空预测；99 条均有 32B Judge，69 条 Judge pass；别名归一后 `31/99 = 31.31%`。

结论：当前 32B Judge/Memory 模块已经合规接入，但在这次配置下没有提升 SimpleVQA 准确率。主要瓶颈是 9B 对 SimpleVQA 视觉实体识别不稳定，以及部分样本空输出。2Wiki 只能比较非空率与轨迹质量，因为本地文件没有 gold。

## 推荐继续运行命令

先跑原始 baseline：

```bash
cd /inspire/qb-ilm2/project/26summer-camp-01/26210300/harness-sii
conda run -n pegp python run_validation.py --dataset 2wiki --mode baseline --max-steps 1 --timeout-sec 180 --max-tokens 512 --disable-tools --out-dir validation_full --resume
conda run -n pegp python run_validation.py --dataset simplevqa --mode baseline --max-steps 1 --timeout-sec 180 --max-tokens 512 --disable-tools --out-dir validation_full --resume
```

再跑带 Judge/记忆的版本：

```bash
cd /inspire/qb-ilm2/project/26summer-camp-01/26210300/harness-sii
conda run -n pegp python run_validation.py --dataset 2wiki --mode evolved --max-steps 1 --timeout-sec 180 --max-tokens 512 --disable-tools --judge --attempts 1 --out-dir validation_full --resume
conda run -n pegp python run_validation.py --dataset simplevqa --mode evolved --max-steps 1 --timeout-sec 180 --max-tokens 512 --disable-tools --judge --attempts 1 --out-dir validation_full --resume
```

并发运行可加：

```bash
--workers 2 --batch-size 2
```

例如：

```bash
cd /inspire/qb-ilm2/project/26summer-camp-01/26210300/harness-sii
conda run -n pegp python run_validation.py --dataset simplevqa --mode evolved --max-steps 1 --timeout-sec 180 --max-tokens 512 --disable-tools --judge --attempts 1 --out-dir validation_full --resume --workers 2 --batch-size 2
```

并发实现说明：

- `--workers N` 只并发 9B 基座模型作答。
- 32B Judge/Memory 在主进程按结果顺序处理，避免并发写记忆，也避免 32B 被用作基座模型。
- `--resume` 可继续已有结果，不重复已完成 index。

如果需要尝试图搜增强 SimpleVQA，可小批量运行：

```bash
cd /inspire/qb-ilm2/project/26summer-camp-01/26210300/harness-sii
conda run -n pegp python run_validation.py --dataset simplevqa --mode baseline --max-steps 3 --timeout-sec 260 --max-tokens 1024 --out-dir validation_tool_full --resume
```

注意：工具模式更慢，建议先用 `--limit 10` 或 `--start N --limit M` 分段验证。
