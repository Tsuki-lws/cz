## 4. 进化代码与模块评分关注点

用户关心评分中的两个核心模块：

### Reflection / 反思模块

评分维度：

- 失败后是否自动触发 Reflection。
- 是否能定位失败原因。
- 是否能自动生成修正策略。
- 是否能把当前任务上的反思结果应用到之后任务。
- 是否体现新增模块相对于基模自带反思能力的增益。

PPT 可强调：

- 失败后不是简单重试，而是先分析失败原因。
- 将失败案例转化为后续可复用策略。
- 可与 memory 或 skill 结合，沉淀为长期经验。

### Memory / 记忆模块

评分维度：

- 是否存在短期或长期记忆。
- 是否结构化存储。
- 是否有记忆更新机制。
- 是否能在后续任务中调用。
- 是否能减少重复错误。
- 是否能保留好经验和坏经验。

PPT 可强调：

- 将历史经验沉淀为结构化知识。
- 后续任务推理前检索相关经验。
- 对失败模式做负例记忆，对成功策略做正例记忆。
